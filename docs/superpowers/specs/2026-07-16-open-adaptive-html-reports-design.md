# Дизайн открытия адаптивных walk-forward HTML-отчётов

Дата: 2026-07-16  
Статус: одобрено пользователем

## Цель

Добавить отдельный сценарий, который одним запуском открывает семь новых HTML-отчётов адаптивной режимной walk-forward стратегии в одном новом окне Google Chrome.

## Границы

Сценарий открывает только:

- `rts/research/adaptive_regime_walk_forward/report.html`;
- `mix/research/adaptive_regime_walk_forward/report.html`;
- `ng/research/adaptive_regime_walk_forward/report.html`;
- `br/research/adaptive_regime_walk_forward/report.html`;
- `gold/research/adaptive_regime_walk_forward/report.html`;
- `si/research/adaptive_regime_walk_forward/report.html`;
- `spyf/research/adaptive_regime_walk_forward/report.html`.

Существующий `html_open.py`, старые `strategy_analysis.html` и отчёт Buhinvest не изменяются и не открываются.

## Архитектура

Создаётся корневой файл `html_open_adaptive_regime.py`. Он определяет корень проекта через `Path(__file__).resolve().parent`, поэтому пути не зависят от текущего рабочего каталога.

Компоненты сценария:

- константа с семью тикерами;
- константа пути `C:/Program Files/Google/Chrome/Application/chrome.exe`;
- функция сбора существующих и отсутствующих отчётов;
- функция формирования одного вызова Chrome с `--new-window`;
- `main()`, печатающий результат и возвращающий код процесса.

## Обработка ошибок

- Отсутствующий отчёт выводится как предупреждение и не мешает открыть остальные.
- Если не найден ни один отчёт, Chrome не запускается, код возврата равен `1`.
- Если отсутствует `chrome.exe`, отчёты не открываются, выводится понятная ошибка, код возврата равен `1`.
- Если найден хотя бы один отчёт и Chrome существует, выполняется один вызов `subprocess.Popen`, код возврата равен `0`.

## Интерфейс

Запуск:

```powershell
python html_open_adaptive_regime.py
```

В верхней docstring нового Python-файла будут русское описание и пример запуска. Все функции получат русские docstring.

## Тестирование

Стандартный `unittest` проверяет:

1. формирование путей для всех семи тикеров;
2. разделение найденных и отсутствующих файлов;
3. единственный вызов Chrome с аргументами `chrome.exe`, `--new-window` и всеми найденными HTML;
4. отсутствие запуска при пустом списке отчётов;
5. код `1` при отсутствующем Chrome;
6. код `0` при наличии Chrome и хотя бы одного отчёта.

Тест передаёт подменяемую функцию запуска процесса через аргумент функции. Реальный Chrome при автоматических тестах не открывается.

## Критерии завершения

- Новый сценарий открывает все доступные walk-forward отчёты одним окном Chrome.
- Частично отсутствующие отчёты не останавливают запуск.
- Существующий `html_open.py` не изменён.
- Автоматические тесты проходят.
- Изменения закоммичены с русским сообщением и отправлены в `origin/main`.
